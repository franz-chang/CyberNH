Case 01 EduMirror

把教育理论转译成 Agent 的价值系统与互动机制

社会动力学学生、教师、家长与同伴如何在互动中形成行为变化。
价值驱动Agent 是否能根据心理需求与社会价值产生理论一致行为。
干预分析系统是否支持教育干预、反事实比较和机制追踪。

Lin J, Yu H, Zeng Y, et al. EduMirror: Modeling Educational Social Dynamics with Value-driven Multi-agent Simulation, arXiv, 2026.

强调 EduMirror 的关键不是“让学生说话”，而是把教育心理理论拆成可执行的价值状态与行动选择。
Case 01 EduMirror · 实验设计

验证逻辑：像不像人、机制是否合理、干预是否符合理论预期

01

系统级验证

覆盖 17 个教育场景，并与 ReAct、BabyAGI、LLMob、JAG-Concordia、D2A 等 baseline 比较。

02

可扩展性测试

在幼儿园场景中改变 agent 数量，评价自然性、一致性、合理性和发展适切性。

03

校园欺凌案例

比较无干预、权威惩戒、支持性个体干预和支持性合作干预。

04

同伴互动案例

测试 SVO 稳定性，并比较团队竞争、教师提醒和公平预教育策略。

行为相似性

模拟学生的行为是否像真实学生，是否自然、连贯、可信。

内部机制

行为是否来自合理的心理需求和社会价值取向，而不是随机生成。

反事实变化

移除价值模型或改变教师干预后，结果是否按教育理论改变。

Lin J, Yu H, Zeng Y, et al. EduMirror, arXiv, 2026.

这一页讲实验逻辑。三步检验尤其重要：行为表现、内部机制、干预后的反事实变化。
Case 01 EduMirror · 结果分析

“理论驱动 + 价值驱动”的优势在于让行为可解释、可消融、可干预

3

类分析问题

  • 行为是否像人：LLM Rater 评估自然性、连贯性、可信性和人类相似度。
  • 机制是否合理：LLM Surveyor 追踪安全感、归属感、自尊与心理健康。
  • 干预是否有效：通过反事实比较分析教师干预和公平教育的影响。

方法启示

理论先行

先拆解教育构念，再映射为 agent 内部状态与行动规则。

状态追踪

不是只看对话文本，还持续观测心理需求和社会价值。

干预比较

同一情境下替换干预策略，观察心理和行为轨迹变化。

边界明确

适合机制探索，不等价于真实教育系统的因果证明。

Lin J, Yu H, Zeng Y, et al. EduMirror, arXiv, 2026.

收束 EduMirror：它的价值在于给教育干预提供一个可控的仿真实验台，而不是直接替代真实实验。
Case 02 Trust Behavior

用受控信任博弈校准 LLM Agent 的微观社会心理行为

研究问题

  • 如果把 LLM Agent 当作“人”的替身做社会模拟，它们的基本信任行为是否可靠。
  • 信任是否来自互惠预期、风险感知或亲社会偏好，而不是任意生成。
  • Agent 输出行动的同时，也输出 Belief、Desire、Intention 以解释决策逻辑。
Trustor

出资方，决定转账金额，表达信任程度。

Trustee

回报方，决定返还金额，体现互惠逻辑。

Jia F, Ye Z, Lai S, et al. Can Large Language Model Agents Simulate Human Trust Behavior? NeurIPS, 2024.

Trust Behavior 是微观校准案例。它不追求开放世界，而是用行为经济学实验把变量控制清楚。
Case 02 Trust Behavior · 实验方法

六类博弈分别拆解信任、慷慨、风险、亲社会偏好和动态互惠

01

Trust Game

衡量初始转账金额，观察 agent 是否表现出信任。

02

Dictator Game

区分“慷慨”与“相信对方会回报我”的互惠预期。

03

MAP / Risky

区分风险感知与信任对象，比较不同风险水平下的 trust rate。

04

Repeated Game

分析多轮信任动态，观察互惠关系是否稳定形成。

基础信任

Agent 是否会主动把资源托付给他人。

互惠预期

转账是否来自“对方会回报我”的信念。

风险感知

Agent 是否根据不确定性调整信任强度。

亲社会偏好

Agent 是否更愿意在人际互动中承担风险。

Jia F, Ye Z, Lai S, et al. Can Large Language Model Agents Simulate Human Trust Behavior? NeurIPS, 2024.

这一页把实验拆成五个维度。重点是信任不是单一数字，而是由互惠、风险、偏好和动态学习共同构成。
Case 02 Trust Behavior · 结果分析

Behavioral Alignment：比较 Agent Trust 与 Human Trust 是否同构

互惠预期

比较 Trust Game 与 Dictator Game 的转账差异，判断信任是否建立在回报预期上。

风险感知

分析 MAP Trust Game 与 Risky Dictator Game 中 trust rate 随风险变化的曲线。

信任动态

通过 Repeated Trust Game 观察多轮转账、返还和比例稳定性。

额外边界检查

作者进一步分析人口属性偏差、人类 / LLM 对象差异、显式指令影响和 CoT 推理影响，以判断 trust behavior 的可操纵性。

Jia F, Ye Z, Lai S, et al. Can Large Language Model Agents Simulate Human Trust Behavior? NeurIPS, 2024.

强调这个案例给社会仿真一个基本前提检查：如果微观信任行为都不稳定，复杂社会仿真的解释力就会被削弱。
Case 03 CompeteAI

餐厅市场作为竞争环境：让微观策略在日级互动中形成宏观现象

仿真场景

  • 一个虚拟小镇中，两个餐厅作为竞争者，约 50 个顾客作为消费者和评价者。
  • 餐厅通过菜单、厨师、广告、定价和经营策略争夺顾客。
  • 顾客根据偏好、预算、健康需求、评论和群体讨论选择餐厅。

核心问题

LLM Agent 能否在竞争环境中形成策略适应，并产生差异化、模仿、马太效应和赢家通吃等市场现象。

Zhao Q, Wang J, Zhang Y, et al. CompeteAI: Understanding the Competition Dynamics in Large Language Model-based Agents, arXiv, 2023.

CompeteAI 是系统涌现案例。研究重点从个体校准转到竞争机制如何在多轮反馈中放大为市场结构。
Case 03 CompeteAI · 实验设计

经营 API、顾客反馈和策略更新形成闭环

15 天餐厅竞争仿真

  • 如果某餐厅退出竞争，仿真提前结束。
  • 设置个体顾客和群体顾客两种条件。
  • 每天餐厅读取账本、顾客评论和竞争对手信息后调整策略。
  • 顾客选择就餐并留下评分、评论和选择理由。
记录变量

顾客流量、菜单变化、广告、评论、菜品评分、顾客选择理由和餐厅收益。

重复运行

观察竞争策略是否稳定出现，以及顾客组织形式是否影响宏观结果。

Zhao Q, Wang J, Zhang Y, et al. CompeteAI, arXiv, 2023.

突出 CompeteAI 的闭环：观察、行动、反馈、更新。它更接近一个简化市场实验室。
Case 03 CompeteAI · 结果分析

从餐厅策略与顾客理由，推断宏观市场结构如何涌现

微观策略

分析餐厅是否从顾客流量、菜品反馈和竞争对手动作中识别竞争态势。

顾客决策

统计核心需求、品牌忠诚、口碑、价格、招牌菜和探索新事物等选择理由。

宏观现象

用流量曲线、菜单相似度、菜品评分和现象频率支持马太效应与赢家通吃分析。

核心发现的解释方式

差异化和模仿形成动态均衡；早期优势经由评论和评分反馈被放大；群体顾客更愿意探索新选择，因此可能削弱赢家通吃。

Zhao Q, Wang J, Zhang Y, et al. CompeteAI, arXiv, 2023.

这一页讲从微观到宏观。CompeteAI 的意义在于展示市场结构不是预设出来，而是从策略和反馈中逐步形成。
Case 04 WarAgent

把国家建模为 Agent，模拟战争如何从多轮外交与动员中爆发

研究问题

  • LLM Agent 是否能较真实地复现历史战略决策和国际关系演化。
  • 某些战争触发事件是否比其他事件更关键。
  • 历史战争是否不可避免，还是在改变条件后可能走向和平。

国家 Agent Profile

领导制度、军事能力、资源状况、历史背景、关键政策和公众士气。

Hua W, Fan L, Li L, et al. War and Peace (WarAgent): LLM-based Multi-Agent Simulation of World Wars, arXiv, 2023.

WarAgent 是宏观历史案例。这里的 Agent 不再是个人或企业，而是国家。
Case 04 WarAgent · 实验设计

从历史复现到反事实注入：检验战争路径对条件变化的敏感性

  • 在一战、二战和战国等历史场景中运行仿真，检验国家 Agent 能否复现结盟、动员和宣战过程。
  • 通过匿名化、反事实知识注入和去匿名化实验，判断行为基于情境推理还是历史记忆。
  • 改变战争触发事件、决策风格和国家 profile 变量，观察战争是否仍会爆发。
A

等待

国家选择观望，推迟外交或军事行动。

B

总动员

内部进入战争准备状态。

C

结盟 / 条约

国家间形成安全承诺或不干涉协议。

D

宣战 / 和平

冲突升级或通过外交走向缓和。

Hua W, Fan L, Li L, et al. WarAgent, arXiv, 2023.

这页强调 WarAgent 的反事实设计。它不是只复现历史，也会改变条件看路径是否改变。
Case 04 WarAgent · 结果分析

结果分析同时需要数值对照、行动解释和反事实比较

历史对照

把仿真结果与真实同盟结构、宣战关系和总动员情况比较；用分区相似度和 Jaccard 相似度衡量重合程度。

人工解释

分析每个国家为什么做出某些行动，例如英法结盟、德奥结盟、美国不干涉或错误宣战。

反事实比较

改变触发事件、决策风格和国家 profile，观察是否只形成冷战、是否更早宣战或是否走向和平。

边界

该方法更适合探索“哪些因素可能改变战争路径”，而不是证明唯一因果或替代历史学解释。

Hua W, Fan L, Li L, et al. WarAgent, arXiv, 2023.

收束 WarAgent：宏观仿真的难点是结果看起来宏大，但仍需要可解释的行动链条和反事实支撑。

Cross-case Comparison

四个案例的共性：都在追求可解释、可验证、可对照的 Agent 仿真

共同方法论

  • 先定义社会机制，再构建 agent 角色、状态和互动规则。
  • 不仅看输出是否合理，还检查内部状态、行为轨迹和宏观结果。
  • 通过 baseline、消融或反事实比较，把“看起来像”转化为“可验证”。
这一页把四个案例放回同一个框架里。共同点是都避免只做故事生成，而是强调验证设计。

Summary

LLM-Agent 社会仿真的关键，不是更像人，而是更可检验

微观校准

Trust Behavior 说明，在把 Agent 用作社会模拟参与者之前，需要先校准基础心理行为。

机制涌现

CompeteAI 展示了如何从个体策略、反馈和重复互动中观察市场结构变化。

应用干预

EduMirror 说明特定领域需要理论驱动建模，才能支持干预设计和反事实分析。

WarAgent 则提醒我们：当仿真对象上升到国家和历史冲突时，模型输出必须接受更强的历史对照、解释审查和反事实检验。

最后给出一句话总结：社会科学仿真的价值不在于生成一个逼真的世界，而在于提供一个可检查机制的实验空间。
